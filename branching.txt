what is branching:
--------------------
Branch = Independent line of development

Default: Main 

Create branch:
-------------------

git branch -b feature1  

git checkout -b feature-login  (Creates the branch and checkout to that branch.)

git branch (this will list all branches present)

 delete a branch:

 git branch -d feature1

Merge Types:
---------------
merge vs rebase..

both are used for merging changes from one branch into another branch..

merge:
----------

it preserves the history of all the commits 
and it create a new commit at the end with details related to both branches..

main 

c1 (xyz)

c2 (xyyzz)


feature (f1) (ABC)

git merge main  (when ur or feature branch)

get merge feature (when ur on main branch)

it will create a new commit and preserver all the history of your both branches..

c1--f1--c2--f2--new_commit  (default merge pattern)


Rebase:
----------
it wont preseve the history it actually overrides the commits data 
and it creates a nice linear history without adding any extra commits..

git rebase main (this u will execute when ur on feature branch)

c1--c2--f1  (This is linear way of doing it..)

if your on main branch and exuecte a rebase againist your feature branch then all the changes of your main
branch will go and sit down to the feature branch which is not expected..
